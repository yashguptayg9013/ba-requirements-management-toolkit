## Change Request Process

1. **Submit** — Requestor completes a Change Request (CR) entry with description and justification
2. **Assess** — BA/PM evaluates impact on scope, schedule, budget, and resources
3. **Review** — Change Control Board (CCB) or designated approver reviews the assessment
4. **Decide** — CR is Approved, Rejected, or Deferred
5. **Implement** — If approved, requirements/baseline documents are updated and re-versioned
6. **Close** — CR marked as closed once implemented and verified

## Status Definitions

| Status | Meaning |
|--------|---------|
| **Submitted** | CR has been logged but not yet assessed |
| **Under Review** | Impact assessment in progress |
| **Approved** | CCB has approved; ready for implementation |
| **Rejected** | CR will not be implemented; reason documented |
| **Deferred** | Valid but postponed to a future phase/release |
| **Implemented** | Change has been made to the system/requirements |
| **Closed** | Change verified and fully completed |
